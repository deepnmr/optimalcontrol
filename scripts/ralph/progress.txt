## Codebase Patterns
- Documentation guides should use runnable Python snippets that import helpers from their defining modules; `optimalcontrol.__init__` intentionally exports only a curated public subset.
- Paper-figure documentation can treat `examples/*.py` as the source of truth: each script exposes `run()`, uses `matplotlib.use("Agg")`, saves a PNG under `examples/output/`, and has a numerical snapshot under `examples/expected/`.
- Build backend: use setuptools (hatchling not available in this environment). Set `[tool.setuptools.packages.find]` with `include = ["optimalcontrol*"]` to avoid setuptools detecting refdoc/ as a top-level package.
- Typecheck: run `mypy optimalcontrol/` from project root; requires at least one .py file in the directory.
- Install: `pip install -e ".[dev]"` installs runtime + dev dependencies in editable mode.
- Tests: use `python3.10 -m pytest` (or `python3 -m pytest`), NOT bare `pytest` — the pytest script fails to pick up the editable install path in this environment. The `-m` flag ensures sys.path includes site-packages and the editable finder is registered.
- Numpy type annotations: mypy strict mode requires `npt.NDArray[np.complex128]` (from `numpy.typing`), NOT bare `ndarray` — bare `ndarray` triggers "Missing type parameters for generic type" errors.
- Distortion helpers live in `optimalcontrol/distortions.py`; keep waveform shape as `(n_steps, n_channels)` with time on axis 0 even though Spinach documents the transposed XY-by-time layout, and have `_deriv` helpers return same-shape local sensitivities rather than full Jacobians.
- Superoperator convention (column-major): L_op(A) = I⊗A = np.kron(I,A), R_op(A) = A^T⊗I = np.kron(A.T,I). PRD text uses row-major labels — ignore and use these formulas.
- scipy.sparse: import csr_matrix at module level; mypy handles it via scipy's inline stubs. Use Union[npt.NDArray[np.complex128], csr_matrix] for functions returning either.
- Spin-system convention: `two_spin_system()` uses spin index 0 as I/source and 1 as S/target, with zero `shifts_hz` for on-resonance paper models and relaxation rates stored in rad/s.
- Hamiltonian builders live in `optimalcontrol/spin_system.py`; convert public `J_hz` and `shifts_hz` values to rad/s with `2*pi`, and use `Iz_i @ Iz_j` for scalar J drift without an extra factor of 2.
- Liouville generators in `spin_system.py` are assembled by converting Hilbert-space Hamiltonians and control operators with `liouvillian_comm()`; control amplitudes multiply dimensionless spin operators and are assumed to already be in rad/s.
- State labels in `optimalcontrol.states` are unnormalised product operators: `I*` labels act on spin 0, `S*` labels act on spin 1, and labels such as `2IzSz` include the literal leading factor of 2; apply `normalise_hs()` before fidelity evaluation.
- Fidelity helpers in `optimalcontrol.states` expect pre-normalised arrays and use `np.vdot(rho_t, rho_f)` as the overlap, matching `Tr(rho_t† @ rho_f)` for matrices while also supporting vectorised states.
- Product-operator basis states such as `Iz` and `2IzSz` are Hilbert-Schmidt orthogonal at the state-construction layer; test transfer efficiency after applying a propagator, not by direct initial-target overlap.
- State hooks in `optimalcontrol.states` accept Hilbert-space rotations for square matrix states (`U @ rho @ U.conj().T`) and Liouville-space propagators for matrix or vectorised states; `dead_time_propagation()` expects a Liouville generator and uses `expm(generator * t_dead)`.
- ROPE analytical helpers live in `optimalcontrol/rope.py`; paper-level functions use Hz-style formulas, require positive `J_hz`, and use `atan2(1.0, n)` for arccot on the physical non-negative branch.
- `rope_g_inphase(n)` assumes equal I/S relative relaxation (`kI/J = kS/J = n`) and returns `rope_g(n) ** 2`; the full JMR expression is the product of separate I-side and S-side ROPE factors.
- ROPE scalar trajectory helpers use dimensionless expectation values; singular ratios return `0.0` for zero numerator and `math.inf` for non-zero numerator over zero denominator.
- Finite-time ROPE solves Appendix B in scaled time `s = pi * J_hz * t`; `rope_j(s, n)` expects that scaled time, while public duration/switching APIs accept and return seconds.
- ROPE waveform sampling uses physical seconds in `rope_waveform()`, but `rope_phase1_control()` and `rope_phase3_control()` use scaled times `pi * J_hz * t` because their signatures do not include `J_hz`.
- ROPE analytical regression tests live in `tests/test_rope.py`; use `numpy.testing.assert_allclose` with the rtol requested by each story.
- CROP analytical helpers live in `optimalcontrol/crop.py`; treat `ka`, `kc`, primed rates, and `J_hz` as same-unit frequency scalars for dimensionless ratios, with `ka >= kc` required for real `zeta`.
- CROP scalar pulse params use Hz units: residual bandwidth `sqrt(ka^2-kc^2)` for amplitude, `-J_hz/2` for the slowly relaxing beta multiplet carrier, and inverse residual bandwidth for the default truncation window.
- CROP waveform sampling uses physical seconds centered on zero over `truncation_window`, and samples Hz-valued amplitude/frequency arrays from `crop_pulse_params()`.
- CROP analytical regression tests live in `tests/test_crop.py`; stored eta values for `(ka,kc,J)=(60,45,100)` and `(110,82.5,100)` are `0.7015664580244381` and `0.5854918065322852`.
- GRAPE control containers live in `optimalcontrol/grape.py`; validate generator dimensions against vector states directly and square matrix states via either Hilbert dimension (`dim`) or Liouville vectorised dimension (`dim^2`).
- GRAPE freeze masks are boolean 2-D arrays shaped as waveform rows by control channels; validation checks the channel count against `len(ControlProblem.operators)`.
- GRAPE propagation treats `ControlProblem.drifts` and `operators` as generator matrices: slice propagators use `expm(generator * pulse_dt)`, with waveform entries scaled by per-channel `pwr_levels`.
- GRAPE forward-state accumulation mirrors state hooks: vectors use `U @ rho`, square matrix states use Hilbert propagation when `U.shape == rho.shape`, otherwise Liouville propagation via `vec`/`unvec` when `U.shape == (rho.size, rho.size)`.
- GRAPE backward-state accumulation returns target-first states `[lambda_N, lambda_N-1, ..., lambda_0]` and applies each slice propagator's adjoint, so `final_fidelity()` compares `fwd_states[-1]` with `bwd_states[0]`.
- `dir_diff_expm(H, dH, dt)` differentiates `expm(-1j*H*dt)`; for existing GRAPE generator matrices, map with `H=1j*generator` and `dH=1j*d_generator` to differentiate `expm(generator*dt)`.
- `grape_gradient()` uses analytical forward sensitivity propagation for vector, Hilbert-matrix, and Liouville-matrix states, then zeroes entries where the freeze mask is true.
- GRAPE gradient regression tests use central finite differences against `grape_xy()` on small deterministic waveforms; Hilbert tests use anti-Hermitian generators, and Liouville tests use `liouvillian_comm()` with vectorised normalised product-operator states.
- `grape_hessian()` uses exact forward second-sensitivity propagation only when flattened waveform parameters are <=50; frozen parameter rows and columns are zeroed.
- `grape_xy()` dispatches `basis == "liouville"` to vectorised density propagation and `basis == "hilbert"` to pure-state vectors; the default dense path preserves the mixed representation behavior.
- GRAPE amplitude/phase adapters use XY waveforms shaped `(n_steps, 2)` with columns X and Y; `xy_to_ampl_phase()` returns 1-D per-step arrays and `ampl_phase_to_xy()` round-trips with `x=A*cos(phi)`, `y=A*sin(phi)`.
- GRAPE curvilinear reparameterisation maps unconstrained waveform values into scalar bounds with `midpoint + half_width * tanh(wfm)`.
- Ensemble expansion lives in `optimalcontrol/ensemble.py`; drift/RF-power axes are converted into single-drift GRAPE problems, and RF ensemble power levels are absorbed into scaled control operators with expanded `pwr_levels` reset to per-channel ones.
- Interpret `ControlProblem.pwr_levels` as ordinary per-channel scaling when its length matches `len(operators)`; treat it as an RF-power ensemble axis only when the length differs from the operator count.
- Ensemble optional axes clear consumed metadata after expansion: offsets add `offset * sum(offset_operators)` to drift generators, and phase-cycle rows are radians that rotate initial states by global or per-state complex phases.
- Correlated ensemble modes are non-combinatorial: `correlated_rho_match()` splits matched source-target pairs, while `correlated_rho_drift()` requires one drift per source-target pair and matches them by index.
- Ensemble backend helpers preserve input order; `joblib_backend()` is optional and falls back to `serial_backend()` when joblib is unavailable.
- Penalty helpers live in `optimalcontrol/penalties.py`; each returns a minimisation-style `(value, gradient)` pair with gradient shaped like the 2-D float64 waveform.
- Penalty conventions: `NS` is elementwise norm-square, `SNS` is elementwise Cartesian spillout above `abs(sample) > limit`, `SNSA` is row-amplitude spillout across channels, and `DNS` uses adjacent time-row finite differences.
- Optimizer helpers live in `optimalcontrol/optimizers.py`; `line_search_cubic()` treats `f` as a maximisation objective, internally applies Wolfe checks to `-f`, and returns `0.0` for non-ascent directions.
- L-BFGS helpers in `optimalcontrol/optimizers.py` store curvature for minimising `-f`: use `s = wfm_new - wfm_old`, `y = grad_old - grad_new`, and fall back to the raw gradient if the two-loop recursion does not produce an ascent direction.
- Newton-Raphson in `optimalcontrol/optimizers.py` works on the exact small-waveform `grape_hessian()` by solving the Newton system for `-f`; regularisation shifts `-H` to positive-definite before solving, and `rfo=True` swaps in an augmented-Hessian level shift.
- Optimizer checkpoints in `optimalcontrol/optimizers.py` are JSON files that always store waveform and fidelity history, and optimizer-driven saves also persist `n_feval` plus L-BFGS memory so resumed runs can match uninterrupted trajectories; function-level `checkpoint_path` overrides `ControlProblem.checkpoint_path`.
- `run_grape()` lives in `optimalcontrol/optimizers.py` and imports `waveform_from_result()` locally to avoid an `optimizers.py` ↔ `io.py` import cycle; it returns the exported `Waveform` plus the raw `OptimResult`.
- Optimizer tests can monkeypatch `optimalcontrol.optimizers.grape_xy` and `grape_gradient` for deterministic toy objectives; monkeypatch `line_search_cubic` to a fixed positive step when checkpoint tests need non-converged partial runs.
- Product-operator matrix-state GRAPE transfer tests should use `basis="dense"`; `basis="hilbert"` is reserved for pure-state vector GRAPE and rejects square matrix states.
- End-to-end two-spin `Iz -> 2IzSz` GRAPE tests can use dense matrix states with drift `-1j * 2*pi*J*Iz_i@Iz_s`, controls `-1j * [Ix, Iy, Sx, Sy]`, HS-normalised product-operator states, and one `pwr_levels` entry per control.
- Waveform I/O lives in `optimalcontrol/io.py`; exported `Waveform.data` is shaped `(n_channels, n_steps)`, while GRAPE internals use `(n_steps, n_channels)`, so transpose only at I/O boundaries such as `waveform_from_result()` or `loaded.data.T`.
- Spectrometer I/O in `optimalcontrol/io.py` is intentionally minimal: Bruker/JCAMP-DX helpers are documented stubs, heterodyne transforms rotate `x/y` channels by `exp(i*2*pi*carrier_hz*t)`, and FAPT import maps frequency/amplitude/phase/time rows to `Waveform` channels `frequency_hz`, `amplitude`, `phase_rad`.
- Benchmark scripts live in `benchmarks/` as standalone `python benchmarks/*.py` entry points that print compact CSV-style timing tables using deterministic toy inputs.
